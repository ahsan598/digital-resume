Lift & Shift Migration of Web Application to AWS Cloud

  • Live Demo
  • Source code

This project involved migrating an on-premises web application to the AWS Cloud using a Lift & Shift approach to improve scalability, performance, and availability with minimal application changes. The solution was architected within a custom VPC, leveraging EC2 for compute, RDS for the database, and S3 + CloudFront for static content delivery.

An Application Load Balancer (ALB) and Auto Scaling Groups (ASG) were configured to ensure high availability and automatic scaling during traffic spikes. IAM policies were implemented for secure access control, while Route 53 handled DNS management and domain routing. The application integrated RabbitMQ for message queuing and Memcached for caching, ensuring faster data access and improved throughput. Infrastructure provisioning was fully automated using Terraform and Ansible, enabling repeatable deployments across environments.

  • Rehosted legacy web application to AWS Cloud with zero code refactoring.
  • Deployed scalable architecture using VPC, EC2, RDS, S3, ALB, and ASG for high availability.
  • Integrated RabbitMQ and Memcached to optimize asynchronous processing and caching performance.
  • Automated provisioning and configuration using Terraform for consistent deployments.
  • Secured infrastructure with fine-grained IAM roles and centralized DNS via Route 53.
